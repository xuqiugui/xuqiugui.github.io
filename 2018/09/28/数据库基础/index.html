数据库基础

发表于 2018-09-28 | 分类于 数据库 ， MySQL

  数据库可能看似与我们很遥远，但其实我们每时每刻都在与数据库打交道。你去银行取钱、通讯录里搜索电话号码、查找微信聊天记录等等，都是与数据库相关。
  我们先来了解一下有关数据库的基本知识。

1、数据库

  • 定义：保存有组织的数据的容器（通常是一个文件或一组文件）

    理解数据库的一种最简单的方式是将其想象为一个文件柜，此文件柜是一个存放数据的物理位置，不管数据是什么以及如何组织的。

  • DBMS：database management system.

    数据库软件、数据库管理系统。它和数据库不是一回事。你使用DBMS管理数据库。接下来要学习的MySQL，就是一种DBMS，你可以用它来管理数据库。

2、表

  • 定义：某种特定类型数据的结构化文件。

    当在文件柜中存放资料的时候，一般是先有个文件夹，然后把相关的资料放进去。文件柜相当于数据库，文件夹相当于表，资料相当于表里的数据。

  • 表名：在同一个数据库中，表名要求唯一。不同的数据库中可以有相同的表名。

用一张图来解释：用户信息表

在数据库中，表就是二维结构，就相当于这张图。

用户信息表：表名，在同一数据库中必须唯一
表：这就是用户信息表，是结构化的，必须是姓名、年龄……体重。存储的数据也必须是特定类型的，姓名必须是字符串、年龄是数字类型……

3、列和数据类型

  • 列：表中的一个字段。所有的表都是由一个或多个列组成的。

    用户信息表中，姓名、年龄、电话……都是字段，这张表就是由这几个字段组成的

  • 数据类型：数据的类型。每个表列都有相应的数据类型，它限制了该列中可以存储的数据。

    姓名和电话的数据类型是字符串，其他的都是数字。数字类型是数字，就不能存储文本

4、行和主键

  • 行：表中的一条记录。表中的数据是按行存储的，行数即是记录的总数。

    用户信息表中张三、李四、王二……所在的行就是一条记录。一条记录对应一个用户的所有信息

  • 主键：一列（或一组列），其值能够唯一区分表中的每个行。

    1.在用户信息表中，电话号码，可以作为主键，因为每个人的电话号码都是不同的，可以用它来代表一个人。姓名、年龄、体重、身高都可以有重复的，就无法来确定唯一的人。

    2.虽然没有主键也可以建表，但是我们应该总是定义主键。主键只要满足以下条件：

    • 任意两行都不具有相同的主键；
    • 每个行都必须有一个主键值（主键值不允许为空值【NULL】）

    3.好习惯：

    • 不更新主键列中的值
    • 不重用主键列中的值
    • 不在主键列中使用可能会更改的值

7、SQL

  • 定义：structured query language.是专门用来与数据库通信的语言。
  • 数据库的优点：
    • SQL简单易学，而且是一种强有力的语言，可以进行非常复杂和高级的数据库操作。
    • 所有数据库的SQL都差不多，学会了一种，几乎能与所有的数据库打交道。但是千万别认为不同数据库中的SQL语法是可以完全移植的，有些还是有很大差别的。
